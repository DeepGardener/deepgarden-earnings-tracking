# DeepGarden 定期财报跟踪 Skill

> 本 Skill 属于 **DeepGarden 投资研究系列**，与 `deepgarden-ipo-research`（DeepGarden 新股研究）并列。
> **DeepGarden 系列投资 Skill 知识库分享由 Deep Garden 出品** —— 欢迎下载、讨论；后续将持续发布相关 Skill，欢迎关注。

Read listed-company periodic reports and produce structured, framework-driven Markdown notes
(Obsidian-friendly). The user supplies the *reading framework* (dimensions + focus points); this
skill provides the deterministic pipeline: archive by company → extract text with page markers →
build a light cross-year summary → dispatch one analysis agent per report → index.

## When to use
- User has a custom periodic-report reading prompt (e.g. 经营 / 资产质量 / 盈利质量 / 值得重视的信息).
- User drops periodic-report PDFs into a folder and asks to organize + read them per that prompt.
- Output target is an Obsidian vault / Markdown notes.

## Workflow

### 1. Set up the archive (by company)
Create under the target vault:
```
定期报告整理/
├─ 00-阅读提示词.md          # paste the user's framework here
├─ 00-README.md
├─ 00-公司索引.md            # top-level index (update as companies added)
├─ 定期报告存档/             # where the user drops raw PDFs
└─ 公司名/
    ├─ 00-公司名索引.md
    ├─ 原始报告/            # COPY (never move) the PDFs here
    └─ 阅读整理/            # per-report MDs go here
```
- Copy PDFs (preserve originals in 定期报告存档). Never delete user files.
- Detect company & report period from filenames (e.g. `联影医疗2024年年度报告.pdf`).

### 2. Extract text with page markers
Run `scripts/extract_reports.py` (depends on `pdfplumber`; install first via `pip install pdfplumber` if missing):
```bash
python scripts/extract_reports.py --src "公司名/原始报告" --out "工作目录/_extract"
```
- Each page is prefixed with `[Pxx]` so analysis can cite exact pages.
- The script also emits `跨年财务摘要.txt` by pulling each annual report's 主要会计数据
  (营收 / 归母净利润 / 加权ROE / 经营现金流净额 / 总资产 / 净资产), giving a compact
  multi-year backbone for trend comparison without reading every full report.

### 3. Dispatch one analysis agent per report
For each report, launch a `general-purpose` subagent (parallel). Give it:
- The framework file `00-阅读提示词.md` (read fully first).
- `跨年财务摘要.txt` (read for trend/同比).
- Its own report's extracted `.txt` (full path).
- Other years' `.txt` paths (only for on-demand cross-year narrative comparison).

Instruct the agent to:
- Use **Grep to locate section line numbers, then Read those regions** (avoid reading whole
  300+ page files). Prioritize: 管理层讨论与分析/董事会报告, 主要会计数据, 合并资产负债表/
  利润表/现金流量表, 公司治理, 股份变动及股东情况, 重要事项(会计变更), 研发投入.
- Cover **every** focus point in the framework; cite pages as `[Pxx]`; tables in Markdown.
- Mark undisclosed items as `（报告未明确披露）` — never fabricate.
- Write `阅读整理/年份+报告类型-阅读整理.md` (e.g. `2024年报-阅读整理.md`).
- End with a 3–5 sentence 快速结论.
- Only create MD files; do not modify/delete any PDF or txt.

### 4. Index and clean up
- Write `公司名/00-公司名索引.md` linking each MD + original PDF, with a financial snapshot
  table and a one-line narrative主线.
- Update top-level `00-公司索引.md`.
- Delete the temp `_extract` directory and helper scripts from the vault (keep the vault clean).

## 参考示例 (Example)

See `references/example-uimages/` for a **real, end-to-end output** generated by this skill for
联影医疗 (688271.SH), following a 22-point framework (经营 / 资产质量 / 盈利质量 / 值得重视的信息):

- `00-联影医疗索引.md` — company index with a cross-year financial snapshot table + 主线 narrative.
- `阅读整理/2023年报-阅读整理.md` … `2026一季报-阅读整理.md` — per-report notes covering **all 4
  framework dimensions**, each focus point cited with `[Pxx]` page marks.

> Original PDFs are omitted (copyright + size); the `[Pxx]` marks refer to the source reports.
> The folder structure mirrors exactly what this skill produces, so installers can preview the result.

pre-flight checklist. Point new users here first.

## Notes / pitfalls
- Large annual reports (300–400 pp, ~300–800 KB text) — never paste full text into the main
  agent context; let subagents read from files on demand.
- Quarterly reports are short and lack annual-only items (full-year ROE trend, multi-year
  董事会报告 continuity) — instruct agents to mark those as `（季报未披露）`.
- Always COPY originals; the 定期报告存档 folder is the user's intake zone — leave it intact.
- 运行环境：使用 WorkBuddy 管理的 Python（终端直接执行 `python` 即可，无需指定绝对路径）。
  脚本依赖 `pdfplumber`，若未安装请先执行 `pip install pdfplumber`（或 `python -m pip install pdfplumber`）；脚本在检测到缺失时会尝试自动安装。

## ★ 与「研报精读」系列双向印证（互证闭环）

本 Skill 是 DeepGarden 投资研究的**系列一（实际业绩）**；`deepgarden-report-deep-read`（公司研报精读）是
**系列二（卖方预测/判断）**。两者在仓库根并列，应互相印证：

- **本系列 → 研报系列**：在 `定期报告整理/<公司>/00-<公司>索引.md` 末尾加一行 wikilink，
  指向 `[[../../研报精读/<公司>/00-<公司>-研报跟踪]]`（研报跟踪 Hub），作为"实际业绩锚"供卖方假设对照。
- **研报系列 → 本系列**：研报精读 MD 含「与定期报告印证」段，并链接本系列索引与具体阅读整理 MD；
  其 Hub 亦汇总"研报假设 vs 定期报告实际"的 ✓兑现 / ⚠️待验 / ✗落空 对照。
- 两处 wikilink 缺一不可，保证"你中有我"的互证闭环。详见 `deepgarden-report-deep-read` 的 Step5 说明。
