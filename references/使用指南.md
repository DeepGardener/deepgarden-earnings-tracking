# DeepGarden 定期财报跟踪 · 使用指南

> **DeepGarden 系列投资 Skill 知识库分享由 Deep Garden 出品。** 欢迎下载、讨论；后续将持续发布相关 Skill，欢迎关注。

> 本文件面向 **安装本 Skill 的使用者**，目的是让你在 5 分钟内准备好输入、照着跑通，并清楚会得到什么产出。

---

## 1. 这个 Skill 帮你做什么

把你**关注的上市公司定期报告**（年报 / 半年报 / 季报）按你自己的**阅读框架（提示词）**逐份读透，生成一套 Obsidian 友好的 Markdown 笔记：

- 每个公司一个文件夹，原始 PDF 与阅读笔记分离归档；
- 每份报告一篇笔记，严格覆盖你框架里的**每一个关注点**；
- 关键数据带 `[Pxx]` **原文页码引用**，方便回查；
- 自动生成公司索引 + 顶层索引，方便跨公司跳转。

---

## 2. 开始前你需要准备什么

| 准备项 | 说明 |
|--------|------|
| ✅ Python | WorkBuddy 自带管理版 Python，终端直接 `python` 即可，**无需指定绝对路径**。 |
| ✅ `pdfplumber` | 提取 PDF 文本用。**未安装时脚本会自动 `pip install`**，无需手动处理。 |
| 📄 阅读提示词 | 你的阅读框架，写入 `00-阅读提示词.md`（格式见 §3.1）。 |
| 📄 定期报告 PDF | 放進 `定期报告存档/`，建议文本型 PDF（见 §6 坑位说明）。 |

> 提示：本 Skill 只做**阅读整理与归档**，不会删除或移动你的任何原始文件（PDF 仅复制）。

## 3. 输入文件准备（重点）

### 3.1 阅读提示词文件：`00-阅读提示词.md`

把你平时读财报最在意的维度写进去即可，结构越清晰，产出越贴合你的习惯。
下面是一个**推荐模板**（与你提供的框架一致），可直接复制修改：

```markdown
# 我的定期报告阅读框架

## 一、经营的关注点
- 董事会工作报告：多年度连续性，看战略是否一贯、目标是否兑现
- 对经营环境的看法（行业趋势 / 政策 / 竞争格局变化）
- 战略方向与差异化（别人没在做的事）
- 杜邦分析：ROE 拆解（净利率 / 周转 / 杠杆），看趋势
- 目标达成习惯（往年说要做的事是否做到）

## 二、资产的质量
- 资产结构最大变化及原因
- 关键比例关系：资产负债率、固定资产比、现金比、存货比、流动/速动比率
- 资本支出与在建工程转固节奏
- 资产周转效率（应收、存货周转等）

## 三、盈利的质量
- 收入与净利润增幅（增速变化）
- 毛利率与三项费用率
- 一次性 / 偶然性收益（政府补助、资产处置等）
- 各业务板块贡献（谁在增长、谁在拖后腿）
- 成本费用结构拆解
- 盈利周期判断（上行 / 下行 / 拐点）
- 净利润 vs 经营现金流（盈利含金量）

## 四、值得重视的信息
- 高管是否连续离职
- 核心管理人员背景
- 激励制度（股权激励 / 员工持股）
- 股权结构与核心人员持股变化
- 会计政策 / 估计变更
- 其他异常现象
```

> 你可以增删条目，Skill 会**逐条覆盖**你写下的每一条；未披露的项会标注「（报告未明确披露）」，绝不编造。

### 3.2 定期报告 PDF 命名规范（强烈建议）

Skill 靠**文件名**自动识别「公司名 + 报告期」，命名越规范，识别越准：

| ✅ 推荐命名 | 说明 |
|------------|------|
| `联影医疗2024年年度报告.pdf` | 公司简称 + 报告期关键词 |
| `贵州茅台2025年半年度报告.pdf` | 半年报 |
| `宁德时代2026年第一季度报告.pdf` | 季报 |

**避坑命名**：
- ❌ 含空格、括号、`-`、`_` 等分隔符易干扰解析 → 建议连写或仅用必要符号；
- ❌ 文件名不含报告期（如 `联影医疗.pdf`）→ 无法判断是哪一期，需手动补；
- ❌ 公司名与日后文件夹名不一致 → 会自动按文件名建文件夹，建议先用你想要的「公司名」命名。

> 若识别失败，最稳妥的办法是**先手动重命名 PDF**，再交给 Skill 处理。

### 3.3 目录结构（按公司归档）

在目标 Obsidian 仓库下建立如下结构（拷贝 PDF，不动原文件）：

```
定期报告整理/
├─ 00-阅读提示词.md          # §3.1 的框架粘进来
├─ 00-README.md
├─ 00-公司索引.md            # 顶层索引（加公司时更新）
├─ 定期报告存档/             # 你丢原始 PDF 的地方
└─ 联影医疗/                # 自动建，公司名即文件夹名
    ├─ 00-联影医疗索引.md
    ├─ 原始报告/            # 从存档 COPY 进来的 PDF
    └─ 阅读整理/            # 每份报告一篇 MD
```

## 4. 运行步骤（四步）

```bash
# 步骤 2：提取文本（带 [Pxx] 页码标记 + 跨年财务摘要）
python scripts/extract_reports.py --src "联影医疗/原始报告" --out "工作目录/_extract"

# 步骤 3：让分析子 agent 逐份按 00-阅读提示词.md 生成阅读 MD（并行）
#   → 每个报告产出 阅读整理/<年份><报告类型>-阅读整理.md
#   → 借助 跨年财务摘要.txt 做同比/趋势对比

# 步骤 4：生成 00-公司名索引.md + 更新 00-公司索引.md，清理临时 _extract
```

> 实际调用由 Skill 编排（含子 agent 派发与跨年摘要），上面命令是给你**理解流程**用的，不必手动一步步敲。

## 5. 你会得到什么（产出）

以联影医疗为例（见 `example-uimages/`）：

| 文件 | 内容 |
|------|------|
| `00-公司名索引.md` | 公司主线叙事 + 跨年财务快照表 + 各报告链接 |
| `阅读整理/2024年报-阅读整理.md` | 单份报告笔记，覆盖 4 大维度全部关注点 |
| `阅读整理/...` | 每份报告一篇，结构一致，便于横向对比 |

**`[Pxx]` 是什么**：提取文本时给每页打的行内标记（如 `[P87]`），阅读笔记里引用数据时会带上它，你回原 PDF 第 87 页即可核对。

## 6. 常见坑 & 注意事项

- **PDF 必须是文本型**：扫描件 / 图片型 PDF 无法提取文字。若提取出来是空白，请先 OCR（如 Adobe / 微信小程序 / 飞书妙记）转成文本 PDF 再跑。
- **年报 vs 季报披露差异**：季报篇幅小、缺全年专项（如全年 ROE 趋势、多年董事会报告连续性）。Skill 已内置处理——这类项会标注「（季报未披露）」，不会硬编。
- **大文件别全量塞进主上下文**：年报常 300–400 页。Skill 用「Grep 定位章节 → Read 片段」的方式精读，不会把整本 PDF 文本灌进对话。
- **公司名/报告期识别失败**：见 §3.2，先手动重命名 PDF。
- **原文件安全**：`定期报告存档/` 是你的「 intake 区」，Skill 只复制，永远保留原样。
